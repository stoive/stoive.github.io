But there's more...

There's a couple of BIG shifts in how the Web is happening in the way it is run behind the scenes, and how users will see it work.

  1. Previously, the W3C would write a standard, and have everyone (supposedly) playing along. But now, the browser vendors innovate first, and the W3C pulls it together into a standard.
  2. Previously, the web would be experienced as a bunch of requests, for and responses of, boring old web pages. But now, a web site can be experienced as an application, like you'd have on the desktop, but immediately and effortlessly distributable, installable, manageable and maintainable.

This has nothing to do with some nifty little tags and a few extra JavaScript functions, but is a shift in the way developers perceive the Web.
